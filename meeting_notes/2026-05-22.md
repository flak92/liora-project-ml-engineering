### Initial data-sourcing approach (yfinance)

- The team initially used the open-source Python library **yfinance** to parse and load market data for S&P 500 and DAX 40 from Yahoo Finance.
- A Python script (`fetch_data.py`) was developed to retrieve data for a configurable time period (default 5 years, up to 10 years), generating CSV files per stock symbol.
- Contributions landed on the GitHub repo to expedite the data-scraping step, prioritizing ML algorithm development over data-scraping infrastructure.

### Alpaca data-source proposal

- Paweł investigated alternative libraries because of yfinance limitations:
- Paweł found out that yfinance data is not supported for backtesting on QuantConnect, while Alpaca is supported, which makes it the more practical choice for continuing the project; this is especially important because the based on amachine learning recommendation engine will high probably require a backtesting engine. Polygon data was also considered, but meaningful historical coverage requires a paid plan. 
- Paweł built a comparison HTML page on GitHub (`data provider choose.html`) comparing Yahoo Finance vs. Alpaca → **~99.5% similarity** on overlapping (ticker, date) pairs (median |relative diff| = 0.000%, p99 = 95%, max = 400% — the tail flagged by split/listing differences, not data corruption).
- **Why Alpaca:**
  - APIs offer more functionalities and simpler access to relevant information.
  - Supports backtesting tools like **QuantConnect**.
  - Alpaca provides professional services to brokers → serious validation pipeline (unlike Yahoo Finance).
- **Trade-off:** Alpaca does **not** cover DAX 40. Team decided to proceed with S&P 500 only — the methodology is market-agnostic, so DAX 40 can be re-added later via another provider if needed.
